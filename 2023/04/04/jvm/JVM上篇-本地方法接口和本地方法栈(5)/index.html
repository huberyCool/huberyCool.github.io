本地方法接口和本地方法栈

1. 本地方法接口

1.1 本地方法是什么？

简单来说，一个Native Method是一个java调用非java代码的接口，该方法的实现由非java语言实现，比如C，这个机制并不是java独有

1.2 本地方法的作用

本地接口的作用是融合不同的编程语言为java所用，他的初衷是融合C/C++程序

1.3 本地方法的定义标识

public class NativeDemo {
    public native void nativeMethodTest();
}

标识符native可以和其他java标识符连用，但是abstract除外

native修饰的方法是有方法体的，而abstract修饰的方法是没有方法体的，所以不能使用abstract修饰

2. 本地方法栈

java虚拟机栈管理java方法的调用，而本地方法栈管理的是本地方法的调用

本地方法栈也是线程私有的

本地方法栈也是支持固定或者可动态扩展的内存大小(在内存溢出方面和虚拟机栈是相同的)

本地方法是C语言实现的

如何使用本地方法

首先在本地方法栈中登记native方法，在执行引擎执行时加载本地方法库

当某个线程调用一个本地方法时，它就不在受虚拟机限制，和虚拟机拥有同样的权限

  • 本地方法可以通过本地方法接口来访问虚拟机内部的运行时数据区
  • 可以直接使用本地处理器中的寄存器
  • 直接从本地内存的堆中分配任意数量的内存
